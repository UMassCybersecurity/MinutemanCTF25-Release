PIO UART 2

Your program must transmit data pulled from the TX FIFO over UART with parity and hardware flow control.

Data should be outputted on pin {{ uart_pin }}(TX), with {{ uart_cycle_rate }} cycles per UART bit.
There should be:

  • 1 start bit
  • 8 data bits
  • 1 parity bit (even parity)
  • 1 stop bit
Additionally, there is an RTS flow control line connected to pin {{ rts_pin }}(CTS).
The RTS line will be held high or low for a random number of cycles in {{ rts_active_range }} and {{ rts_inactive_range }} respectively.

Your program must respect the RTS line and only transmit when it is high. (if a transmission is started and the RTS line is deasserted, the transmission is allowed to finish)

The goal is to correctly transmit {{ uart_valid_bytes_count }} bytes from the TX FIFO to get the flag. The emulator will put a new byte into the FIFO as soon as it is empty.
